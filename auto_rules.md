# Reguły automatyczne – SIEDZI!

> Zasady pracy z repozytorium Git, obowiązujące w każdej sesji.

## Git – na początku sesji

**Jeśli repozytorium Git nie istnieje** (brak `.git`):

- **Zasygnalizuj** przed rozpoczęciem pracy nad kolejnymi zmianami
- Użytkownik może wtedy zdecydować o inicjalizacji repo lub kontynuacji bez tego

## Git – na końcu sesji

**Jeśli repozytorium Git istnieje**:

- Po zakończeniu sesji wykonaj **commit** i **push**
- Commit: opisowy komunikat odzwierciedlający wprowadzone zmiany
- Push: wypchnięcie na zdalne repozytorium (np. `origin main`)
